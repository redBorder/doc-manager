6.9. Tipos de sensores y configuración de dispositivos

En la sección de sensores de un Manager de Redborder, el usuario puede encontrar los diferentes tipos de sensores, cada uno realiza una función diferente:
  • Flujo: Para el monitoreo y la supervisión de la información de tráfico recivida.
  • MSE: Para proporcionar servicios relacionados con la movilidad.
  • Meraki: Sensor específico diseñado para integrar la solución de Meraki (Cisco).
  • Aruba ALE: Para recopilar datos de dispositivos móviles habilitados para redes Wi-Fi (protegiendo la privacidad personal).
  • Dispositivo de monitor: Para mantener el control total sobre el estado de la infraestructura.
  • Twitter: Diseñado para recopilar actividad del uso de Twitter.
  • Vault: Permite visualizar y analizar los logs recibidos de los diferentes servicios y dispositivos.
  • Mail Gateway: Sensor por el cual pasa el tráfico de correo (SMTP).
  • Claim Sensor: Permite vincular el manager con el client proxy, que es un software centralizado que recopila datos de petición realizada por un cliente y los envía al manager.

6.9.1. Monitor

El sensor del dispositivo monitor es uno de los últimos desarrollos que viene con la versión 80-6 de Redborder. Se puede usar para monitorear cualquier dispositivo con una dirección IP que admita snmp, redfish o ipmi. Al crear o actualizar el sensor del dispositivo monitor, se deben definir las credenciales para los protocolos correspondientes. En caso de que no se defina una cadena de comunidad para snmp, se utilizará el "público" predeterminado.
Una vez que el usuario ha iniciado sesión en el dashboard del Manager el sensor se puede agregar usando el menú Sensores > Añadir un Sensor y luego haciendo clic en ‘Monitor’.
Se le puede solicitar algunos parámetros obligatorios y opcionales..
  • Nombre único
  • La dirección IP de la máquina que el usuario desea monitorear
  • El protocolo que el usuario usará para la monitorización
  • Las credenciales de la máquina para acceder al protocolo seleccionado.
  • La cadena de comunidad en caso de que el usuario use el protocolo snmp
Presione el botón "Guardar" para aplicar la configuración
