8.3.4. Radius

El servicio de autenticación remota telefónica de usuario (RADIUS) es un protocolo de red que opera en el puerto 1812, el cual proporciona administración centralizada de autenticación, autorización y contabilidad (AAA o Triple A) para los usuarios que se conectan y utilizan un servicio de red. La clase RadiusProcessor pertenece al módulo rb-Samza-bi.
RADIUS es un protocolo de cliente/servidor que se ejecuta en la capa de aplicación y puede usar TCP o UDP como transporte. Los servidores de acceso a la red, las puertas de enlace que controlan el acceso a una red, generalmente contienen un componente de cliente RADIUS que se comunica con el servidor RADIUS. RADIUS es a menudo el back-end de elección para la autenticación 802.1X también.
En un manager de redborder, al activar las opciones avanzadas en Configuración general, es posible acceder a una sección llamada Radius Accounting y Radius Proxy Authentication.
Opciones Generales: sección de Radius Accounting y Radius Proxy Authentication

8.3.4.1. Radius authentication server

Este apartado permite activar un servidor de autenticación Radius. El usuario debe indicar la IP del Host y contraseña.
Añadir un servidor de Radius Accounting: Es posible añadir un nuevo Servidor Radius clicando sobre la pestaña +Add, a continuación, introducir la IP y la contraseña compartida. Pulse en Update para guardar los cambios introducidos.
Eliminar un servidor de Radius Accounting: Clicar en el icono Eliminar que aparece junto a la información del servidor Radius para descartarlo.
