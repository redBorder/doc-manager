8.2. Operaciones

Hay algunas operaciones importantes que deben ejecutarse para realizar configuraciones avanzadas en Redborder.
  • motd: Motd muestra información general sobre el nodo y el clúster
    Motd: salida
  • rb_get_managers.rb: Muestra información sobre nodos en el clúster
    rb_get_managers.rb: salida
  • rb_get_services.sh: Obtenga todo el estado de los servicios en el nodo. Utilizado con la opción "all" muestra información para todos los nodos. Si el servicio opcional también se añade, solo se muestra el estado de este servicio
    rb_get_services.sh: salida
  • rb_set_service.rb: Permite habilitar o deshabilitar un servicio.
    rb_set_service.rb: uso
  • rb_service: Permite trabajar con un servicio. Puede iniciar, detener y reiniciar servicios. Se puede usar con servicios como postgresql, riak, hadoop, zookeeper, druid, kafka, chef y más.
    rb_service: uso
  • rb_set_topics.rb: Asignar el destino del tema, que puede ser samza o realtime. Los servicios correspondientes deben iniciarse para mantener el funcionamiento correcto de la plataforma Redborder.
    rb_set_topics.rb: uso
  • rb_cluster_configure.rb: Obtenga el estado de todos los servicios para todos los nodos. Permite cambiar la configuración de todo el clúster.
    rb_cluster_configure.rb: uso
  • rb_manager_ssh.sh: Este comando se usa para mover o ejecutar otro comando en otro nodo.
    rb_manager_ssh.sh: uso
  • rb_manager_scp.sh: Este script se usa para copiar un archivo desde el Manager donde se está iniciando a todos los demás nodos o una lista de nodos.
    rb_manager_scp.sh: uso
  • rb_cluster_reboot.sh: Solicita el reinicio de todos los nodos del clúster.
    rb_cluster_reboot.sh: uso
  • rb_cluster_update.sh: Se utiliza para actualizar el Manager a una nueva versión lanzada.
    rb_cluster_update: uso
  • rb_get_tasks.rb: Obtiene información sobre el servicio de indexación.
    rb_get_tasks.rb: uso
  • rb_wakeup_chef: Inicia el cliente de chef en todos los nodos.
    rb_wakeup_chef: uso
