⁠6.9.2.4.2. Sensors: configuración individual de los sensores Mail Gateway
Para editar la configuración de cualquiera de los sensores Mail Gateway a nivel individual el usuario debe dirigirse a la sección Sensors situada en el lado derecho del menú de cabecera. A continuación, deberá seleccionar el sensor que desea editar.
En la vista árbol, los sensores Mail Gateway vienen acompañados de un icono identificativo. También puede buscar el sensor en el listado que se muestra desde View -> Gateway Sensors.
Sensors: vías para la edición de un sensor
Listado de sensores Mail Gateway: Sensors -> View -> Gateway Sensors
Dentro del menú que se despliega encontramos las siguientes opciones:
  • Edit: permite la edición de la configuración de un sensor a nivel individual.
  • Monitors
  • Quarantine: listado de los correos que se encuentran en cuarentena.
  • Filter Policies: configuración y asignación de Filter Policies para cada sensor.
Editar sensor Mail Gateway
Para comenzar la edición de los parámetros del sensor seleccionar Edit.
En la interfaz de edición del sensor se introducirá la siguiente información. Para guardar la configuración del sensor, hacer clic en el botón Update.

Nota

Los campos nombre, IP y versión son no editables, ya que esta información se introduce en el proceso de configuración y registro del sensor.
⁠6.9.2.4.2.1. Gateway Settings
En este apartado el usuario puede definir el comportamiento del sensor en base a los valores que introduzca en cada uno de los campos que se muestran en él.
  • Relay Host: Especificar la IP del servidor de correo final.
  • Message Size Limit: Definir el tamaño máximo de los mensajes que se pueden recibir.
  • Max Paralell Process: Indicar el número máximo de procesos en paralelo que trata el Mail Gateway.
  • API timeout (seconds): Definir el tiempo máximo de espera para la obtención de un score válido por parte de la caché remota.
  • Email detained timeout (seconds): En este campo se indica el tiempo máximo de espera para la obtención de un score válido por parte de la caché remota. Superado este tiempo el fichero se dejará pasar.
  • Quarantine timeout (days): Configurar los días que han de pasar desde que un fichero identificado como Malware es bloqueado y retenido hasta que se define una acción para gestionarlo. Si no se realiza ninguna acción o se supera el tiempo indicado el fichero es descartado.
  • Cache Sync (hours) (minutes): Indicar el tiempo máximo que debe pasar para la sincronización de la caché local y remota. Valor en horas y en minutos.
  • Maximum number of stored log files: Delimitar el tiempo máximo que el fichero permancerá almacenado en el histórico de correos.

⁠6.9.2.4.2.2. Relay Domains
En este apartado el usuario debe definir qué dominio o dominios son los que van a estar gestionados por el sensor.
Aquí se visualiza un listado de los dominios que ya han sido agregados. El usuario puede eliminar en cualquier momento aquellos dominios en desuso o que considere que no deben ser gestionados por el sensor.
Listado de Relay Domains
Para añadir un nuevo dominio basta con hacer clic en la en la pestaña +Add Domains e introducir el dominio que se desea asociar, haga clic en Update para guardar la información.
